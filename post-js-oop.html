[JS] Hướng đối tượng trong JavaScript: Có giống như Java?

"Hồi mới học Java xong qua học JS, mình kiểu: 'Ơ, sao Class trong này nhìn quen thế?'. Nhưng đào sâu hơn một chút, mình nhận ra đó chỉ là 'vỏ bọc' thôi. JavaScript thực chất có một linh hồn rất khác biệt gọi là Prototype."

Nhiều người thường nói Java và JavaScript giống nhau như 'Cá' và 'Cá ngựa'. Điều này đặc biệt đúng khi chúng ta nói về Lập trình hướng đối tượng (OOP). Cùng mình bóc tách xem chúng khác nhau thế nào nhé!

1. Class trong JS: Chiếc "máy ảnh" Filter

Trong Java, Class là một bản thiết kế (blueprint) cực kỳ nghiêm ngặt. Trong JavaScript, từ khóa class thực chất chỉ là "Syntactic Sugar" (cú pháp cho đẹp). Bên dưới lớp vỏ đó, JS vẫn sử dụng cơ chế Prototype-based.

// Khai báo Class trong JS (ES6)
class Mobile {
    constructor(brand, color) {
        this.brand = brand;
        this.color = color;
    }

    showInfo() {
        return `Đây là điện thoại ${this.brand} màu ${this.color}`;
    }
}

const myPhone = new Mobile("iPhone", "Titan");

2. Tính kế thừa: Extends và Super

Cũng giống như Java, JS dùng extends để kế thừa. Tuy nhiên, sự linh hoạt của JS cho phép bạn thêm thuộc tính hoặc phương thức vào đối tượng ngay cả khi nó đã được khởi tạo – điều mà Java "phát điên" nếu bạn thử làm.

class SmartPhone extends Mobile {
    constructor(brand, color, has5G) {
        super(brand, color); // Gọi constructor của cha
        this.has5G = has5G;
    }
}

3. So sánh nhanh: JavaScript vs Java

Đặc điểm Java (Class-based) JavaScript (Prototype-based)
Định nghĩa Nghiêm ngặt, phải có Class mới có Object. Linh hoạt, Object có thể tạo trực tiếp.
Tính kế thừa Kế thừa theo thứ bậc Class. Kế thừa qua chuỗi Prototype.
Kiểu dữ liệu Strongly typed (Kiểu dữ liệu tĩnh). Loosely typed (Kiểu dữ liệu động).

Tại sao chúng ta vẫn cần OOP trong JS?

Dù JS là ngôn ngữ đa hình thái, việc áp dụng OOP giúp code của bạn:

  • Dễ quản lý: Gom nhóm các thuộc tính và hành động liên quan vào một chỗ.
  • Tái sử dụng: Viết một lần, dùng ở nhiều nơi qua kế thừa.
  • Sạch sẽ: Giảm thiểu sự hỗn loạn khi dự án web của bạn phình to.

Lời kết

Đừng để cú pháp giống nhau làm bạn lầm tưởng. Java là một "kỷ luật viên" nghiêm túc, còn JavaScript là một "nghệ sĩ" tự do. Hiểu rõ bản chất Prototype phía sau Class trong JS sẽ giúp bạn làm chủ ngôn ngữ này một cách thực thụ.
