JDBC và MySQL: "Cây cầu" nối liền Java với Cơ sở dữ liệu

"Làm đồ án quản lý mà không có Database thì giống như nấu cơm mà không có gạo vậy. Nhưng mình thề, cái cảm giác lần đầu chạy code mà nó báo 'Connection Success' nó sướng hơn cả lúc nhận điểm 10!"

Chào các bạn, JDBC là một thư viện của Java giúp chúng ta thực hiện các câu lệnh SQL ngay trong mã nguồn Java. Nghe thì đơn giản, nhưng để 'cây cầu' này hoạt động trơn tru thì có rất nhiều chi tiết nhỏ cần lưu ý.

1. Công thức của một Connection String chuẩn

Nhiều bạn thường copy nguyên xi trên mạng về rồi chạy không được vì sai thông tin. Đây là cấu trúc mình luôn dùng:

String url = "jdbc:mysql://localhost:3306/ten_database";
String user = "root";
String password = ""; // Thường XAMPP để trống, MySQL Installer thì có pass

Connection conn = DriverManager.getConnection(url, user, password);

2. Những "hố tử thần" và cách né

Lỗi 1: ClassNotFoundException
Đây là lỗi do bạn quên chưa "vứt" file Driver (mysql-connector-java.jar) vào thư viện của dự án.
Cách sửa: Download file .jar về, chuột phải vào Project -> Build Path -> Add External Archives.
Lỗi 2: Timezone (Múi giờ)
Các bản MySQL mới thường bắt lỗi múi giờ. Nếu gặp lỗi, hãy thêm đoạn này vào sau tên database trong URL:
?useLegacyDatetimeCode=false&serverTimezone=UTC
Lỗi 3: Quên đóng kết nối
Đây là lỗi 'vô hình'. Code vẫn chạy nhưng chạy một lúc là Database bị treo.
Cách sửa: Luôn sử dụng try-with-resources để Java tự động đóng kết nối cho bạn.

3. Code mẫu kết nối "sạch"

Thay vì viết code tràn lan, mình thường đóng gói nó vào một Class DBContext để tái sử dụng ở mọi nơi:

public class DBContext {
    public Connection getConnection() throws Exception {
        String url = "jdbc:mysql://localhost:3306/LibraryDB";
        String user = "root";
        String password = "123";
        Class.forName("com.mysql.cj.jdbc.Driver");
        return DriverManager.getConnection(url, user, password);
    }
}

Lời khuyên cho các bạn làm đồ án

Hãy luôn kiểm tra kết nối Database đầu tiên trước khi viết các chức năng Thêm/Xóa/Sửa. Nếu "cây cầu" chưa vững, đừng vội xây nhà trên đó. Và nhớ nhé, mật khẩu database đừng bao giờ để lộ lên GitHub nếu đó là dự án thực tế!
