Hệ thống Kiến thức Lập trình Mạng

Mục tiêu: Tài liệu này tổng hợp toàn bộ kiến thức chuyên sâu về Mạng máy tính mà tôi đã tích lũy. Từ nền tảng vật lý, các giao thức truyền tải, đến kỹ thuật lập trình Socket và bảo mật hệ thống. Đây là cơ sở để tôi xây dựng các ứng dụng Back-end hiệu năng cao.

I. Nền tảng Mạng máy tính

1. So sánh Mô hình OSI và TCP/IP

Hiểu sự tương quan giữa mô hình lý thuyết (OSI) và mô hình thực tế (TCP/IP) giúp việc debug lỗi mạng chính xác hơn.

Tầng OSI (7 Layer) Tầng TCP/IP (4 Layer) Chức năng chính & Giao thức
7. Application
6. Presentation
5. Session		 Application Giao tiếp với người dùng cuối.
(HTTP, FTP, SMTP, DNS)
4. Transport Transport Truyền tải dữ liệu, kiểm soát luồng.
(TCP, UDP)
3. Network Internet Định tuyến đường đi gói tin (Routing).
(IP, ICMP, ARP)
2. Data Link
1. Physical		 Network Access Truyền bit qua môi trường vật lý.
(Ethernet, Wi-Fi, MAC Address)

2. Địa chỉ & Định tuyến

  • IP Address: Định danh thiết bị trên mạng (IPv4: 32-bit, IPv6: 128-bit).
  • Subnet Mask: Xác định phần mạng (Network ID) và phần máy (Host ID).
  • DNS: "Danh bạ" của Internet, chuyển đổi tên miền (google.com) thành IP.
  • Routing Table: Bản đồ chỉ đường cho Router.
  • Giao thức định tuyến: OSPF (Link-state), RIP (Distance-vector).
  • Gateway: Cổng ra để kết nối với mạng bên ngoài (Internet).

II. Các Giao thức Mạng cốt lõi

TCP (Transmission Control Protocol)

  • Đặc điểm: Hướng kết nối, Tin cậy.
  • Cơ chế: 3-way Handshake, Retransmission.
  • Ứng dụng: Web (HTTP), Email, File Transfer.

UDP (User Datagram Protocol)

  • Đặc điểm: Không kết nối, Nhanh.
  • Cơ chế: Gửi không đảm bảo, không sắp xếp.
  • Ứng dụng: Game Online, Video Streaming, VoIP.

Giao thức Ứng dụng (Application Layer)

  • HTTP/HTTPS: Giao thức của Web. HTTPS sử dụng TLS/SSL để mã hóa.
  • WebSocket: Giao thức truyền tin 2 chiều thời gian thực.
  • SSH: Giao thức quản trị Server từ xa an toàn.

III. Kỹ thuật Lập trình Socket

Socket là giao diện lập trình ứng dụng (API) mạng, cho phép hai tiến trình giao tiếp với nhau.

1. Quy trình hoạt động (Workflow)

Server socket() → bind() → listen() → accept() → read()/write() → close()
Client socket() → connect() → write()/read() → close()

2. Kiến thức nâng cao về I/O

  • Blocking I/O: Chương trình dừng lại chờ dữ liệu (Mặc định).
  • Non-blocking I/O: Chương trình không dừng, kiểm tra liên tục (Polling).
  • I/O Multiplexing: Sử dụng select, poll, epoll để quản lý hàng ngàn kết nối.

3. Ví dụ Code Python (TCP Server)

import socket

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.bind(('localhost', 8080))
server.listen(5)
print("Server đang lắng nghe tại port 8080...")

while True:
    client_socket, addr = server.accept()
    print(f"Đã kết nối với {addr}")
    msg = client_socket.recv(1024).decode('utf-8')
    print(f"Client gửi: {msg}")
    client_socket.close()

IV. Bảo mật & Kiến thức Nâng cao

Bảo mật Mạng (Network Security)

  • Mã hóa (Encryption): AES (Đối xứng), RSA (Bất đối xứng).
  • Firewall: Hệ thống lọc gói tin dựa trên luật (Rules).
  • Chống tấn công: DDoS, SQL Injection, XSS, MITM.

Hệ thống Phân tán (Distributed System)

  • Load Balancing: Cân bằng tải (Nginx, HAProxy).
  • RESTful API: Chuẩn thiết kế API dựa trên HTTP Methods.
  • CDN: Mạng phân phối nội dung toàn cầu.
