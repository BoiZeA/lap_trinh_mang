[Java] Game bóng nảy: Bài học đầu đời về Đồ họa và Vật lý

"Bạn có bao giờ thắc mắc làm sao những nhân vật trong game có thể di chuyển mượt mà trên màn hình không? Hồi mới học Java Swing, mình chỉ biết kéo thả nút bấm (Button) hay ô nhập liệu (TextField). Nhưng khi làm bài tập 'Bouncing Ball', mình mới thực sự hiểu về linh hồn của đồ họa máy tính."

Trong bài viết này, mình sẽ chia sẻ về cách 'vẽ' bằng code và logic va chạm vật lý đơn giản nhất mà bất kỳ lập trình viên Java nào cũng nên trải nghiệm qua.

1. Phương thức paintComponent – Tấm toan của họa sĩ

Trong Java Swing, mọi việc vẽ vời đều diễn ra bên trong phương thức paintComponent của một JPanel. Hãy tưởng tượng nó như một tấm toan trắng, và đối tượng Graphics g chính là cây cọ vẽ của bạn.

@Override
protected void paintComponent(Graphics g) {
    super.paintComponent(g); // Xóa hình cũ trước khi vẽ hình mới
    g.setColor(Color.RED);
    g.fillOval(x, y, 30, 30); // Vẽ quả bóng tại tọa độ (x, y)
}

2. Game Loop: Bí mật của sự chuyển động

Quả bóng không tự di chuyển. Nó "trông có vẻ" di chuyển là vì chúng ta vẽ nó ở vị trí mới liên tục hàng chục lần mỗi giây. Chúng ta cần một Timer để cập nhật tọa độ và gọi hàm repaint().

3. Va chạm vật lý: Khi quả bóng "biết" bật lại

Đây là phần thú vị nhất. Làm sao quả bóng biết khi nào chạm tường để nảy lại? Chúng ta chỉ cần một vài câu lệnh if đơn giản để kiểm tra tọa độ so với kích thước khung hình.

Logic va chạm:
- Nếu x < 0 hoặc x + đường_kính > chiều_rộng: Đổi chiều vận tốc X (vx = -vx).
- Tương tự với trục Y để xử lý cạnh trên và cạnh dưới. 
public void update() {
    x += vx;
    y += vy;

    // Kiểm tra va chạm tường trái/phải
    if (x < 0 || x + 30 > getWidth()) {
        vx = -vx;
    }
    // Kiểm tra va chạm tường trên/dưới
    if (y < 0 || y + 30 > getHeight()) {
        vy = -vy;
    }
}
